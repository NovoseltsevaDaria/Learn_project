# Лабораторная работа №2: Разделение приложения на Frontend и Backend

**Тема:** Создание архитектуры клиент-сервер с использованием микросервисного подхода.
**Стек технологий:** APIFlask (Backend API), Flask (Frontend/Web-интерфейс).  
**Цель работы:** Научиться разделять логику приложения, реализовать взаимодействие между сервисами через HTTP (REST API) и освоить основы работы с APIFlask.

## Описание проекта)))

Проект демонстрирует принцип разделения ответственности:

- **Backend-сервис (APIFlask):** Предоставляет RESTful API для работы с данными (CRUD операции). Возвращает JSON.
- **Frontend-сервис (Flask):** Отвечает за отображение HTML-страниц. Взаимодействует с пользователем, отправляет запросы к Backend через `requests` библиотеку и рендерит полученные данные.
